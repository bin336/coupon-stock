## 二、已确定·待开发 → 已实现（同批 v3.36，已提交 `bef9dfc`，等上 NAS）

> 用户选定并实现完成，尚未部署到 NAS。

### 1）重复录入拦截 ✅ 已实现
- **做什么**：券入库前，按 **券号**（精确）和 **图片 MD5 指纹**（上传截图内容哈希）与现有**未售**券比对；命中相似/重复时后端返回疑似列表，前端弹窗「疑似重复录入，确认仍要录入？」，确认后带 `force=1` 强制录入（单条/批量均支持）。
- **为什么提效/防错**：每日 100+ 张截图，极易把同一张券录两次，直接导致库存虚高、对账出错。前端提交前 + 后端入库前双重把关。
- **注意**：已售出的券不触发拦截（允许回购同券号）；无券号时仅按图片指纹比对。

### 2）今日运营卡片 ✅ 已实现
- **做什么**：首页新增一张「今日运营」卡片，聚合：今日录入张数/面值、今日售出张数/面值、今日结算额（扣 1.6% 后）、当前库存总值、将过期（7 天内）张数。
- **为什么提效**：开局即有全局视图，不用在报表/过期页之间来回切，优先处理将过期券。
- **注意**：新增券的 `created_at` 改存本地时间，确保每日统计按中国时区正确分界（历史数据按 UTC 存，仅边界 8 小时可能有微小偏差）。

### 3）多图批量录入·紧凑卡片 ✅ 已实现（`27f3aa9`）
- **做什么**：批量弹窗里每张图从「一上来就铺开 8 个输入框」改成**紧凑卡片**——默认只显示缩略图 + 一行识别摘要（商家 · 面值 · 券号 · 过期 · 张数）+ 识别状态（🤖 识别中… / ✓ 识别 N 项 / 未识别请手填），点「展开」才显示完整输入框；任一字段改动实时刷新顶部摘要行，缺商家名时摘要变红提醒。
- **为什么提效/好用**：一次选 10+ 张截图时，旧版会堆出上百个输入框、页面极长极挤、滚动找券很累；新版一眼扫完所有券的识别要点，只把需要纠错的卡片展开，录入动线短很多。
- **配套**：沿用 `80944cc` 的 OCR 提速，每张图「🤖 识别中…」进度照常显示；「通用填写-应用到全部」后摘要同步刷新。

## 五、注意事项
- gzip 只解决**传输体积 + 重复打开**的慢；若慢来自手机流量走 Tailscale 中继（DERP）的底层带宽，代码层面无解，只能连 WiFi 或等信号好。
- 数据全在 `./data`（含二维码截图），重建容器不丢。
- 外网 HTTP 明文不安全，条件允许建议加反向代理 + HTTPS（见《使用文档》第九节）。
