# 券库 · 云主机部署指南（比 NAS 更稳的 7×24）

> 适用版本：v3.35+
> 适用人群：想把「券库」跑在云服务器上、告别家里断电/断网/公网 IP 变动的人。
> 前置：你已经有一台**装了 Docker 的 Linux 云主机**（本文会教你买 + 装）。

---

## 一、为什么选云主机（对比 NAS）

| 维度 | 家里 NAS | 云主机 |
|------|----------|--------|
| 公网 IP | 多半没有，要靠 DDNS + 路由器映射，常被运营商回收 | **固定公网 IP，永不变** |
| 断电/断网 | 家里一断就登不上 | 厂商双路供电，SLA 99.9%+ |
| 外网访问 | 要装 Tailscale 打洞 | 直接 `http://公网IP:3000` 就能用 |
| 成本 | 一次性买机器（贵），电费另算 | **约 600~1500 元/年**（续费/常规价，首年抢秒杀可低至几十元） |

> 结论：云主机当「运行服务」，你现有的 NAS 当「备份盘」，**比纯 NAS 更稳，也更省心**。

## 二、选哪家（2025–2026 行情，价格随活动大幅波动）

> ⚠️ **价格重要说明**：下面「首年促销价」是**新人账号 + 大促/秒杀**（双11、618、官网新人专享）才能拿到的价，不是人人随时能买到的常规价。「续费/常规包年价」才是你之后每年大概率要付的钱——**两者能差十几倍**。之前版本把促销价写得太前置、续费价也写低了，此处已更正。

| 厂商 | 机型 | 配置 | 首年促销价（新人/大促） | 续费/常规包年价 | 备注 |
|------|------|------|------------------------|----------------|------|
| 阿里云 | 经济型 e / 轻量 | 2核2G | 38–99 元 | ~600–1000 元 | 99 元 e 实例常可续费同价但限量限时；常规 ECS 偏贵 |
| 腾讯云 | 轻量 | 2核2G / 3M | 38–68 元 | ~600–700 元 | 部分 SKU「续费同价」；需蹲秒杀 |
| 腾讯云 | 轻量 | 2核4G / 6M | ~70–176 元/年（三年均摊更低） | 部分可续费同价 | 性价比高，可一步到位 |
| 华为云 / 京东云 / UCloud | 2核2G~4G | ~几十–200 元 | ~300–800 元 | 看首单折扣力度 |

**现实预算建议**：
- **想稳定可预期**：按「续费/常规包年价」算，2核2G 约 **600–1000 元/年**，2核4G 约 **1000–1800 元/年**。
- **想压到 100 元/年以内**：只能靠「新人账号 + 大促秒杀 + 抢到续费同价 SKU」，且每年可能要换账号/蹲活动，**不稳定，不推荐长期依赖**。
- **按量付费**（不用不扣钱）折合更高，约 **1500 元+/年** 等效，不适合常驻服务。

> **配置建议：2核2G 足够**（券库是 Node + sql.js 纯 JS，无外部数据库，几百~几千条券只占一两百 MB 内存）。想顺手多跑点东西选 2核4G。
> **系统选 Ubuntu 22.04 或 Debian 12**；地域选离你近的（如广州 / 上海）；带宽 3~5M 够用。

## 三、整体思路（一句话）

> 买云主机 → SSH 连上装 Docker → 把项目传上去 → `docker compose up -d --build` → 控制台开 3000 端口 → 手机浏览器开 `http://公网IP:3000` 就能用。

数据（券 + 截图）存在云主机磁盘的 `./data` 目录里，容器删了重建也不丢（和 NAS 一样靠卷挂载）。

## 四、买完后的初始配置（SSH 连上主机）

买完云主机，厂商控制台会给一个**公网 IP** 和**初始密码**，用 SSH 连上去（电脑终端跑）：

```bash
ssh root@你的公网IP
```

### 1) 装 Docker（一键脚本，最稳）
```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER   # 允许当前用户免 sudo 跑 docker（退出重登生效）
```
> 备选（Ubuntu/Debian 自带源）：`sudo apt-get update && sudo apt-get install -y docker.io docker-compose-plugin`
> 验证：`docker --version` 和 `docker compose version` 都出版本号即成功。

### 2) 放行端口（两步都要做）
- **云平台「安全组」**：进厂商控制台 → 找到这台主机的安全组 → 入站规则**允许 TCP 3000**（来源 `0.0.0.0/0`；想更狠可只放你自己的 IP）。
  - 如果用下面的 Tailscale 方案，这步可以**不做**（3000 不暴露公网）。
- **系统防火墙**（若开了 ufw）：`sudo ufw allow 3000`（或 Tailscale 方案下 `sudo ufw allow 22` 仅留 SSH）。

## 五、部署项目

### 方式 A：直接 git 拉（需主机装 git）
```bash
cd ~
git clone https://github.com/bin336/coupon-stock.git
cd coupon-stock
```

### 方式 B：从预览/电脑传 zip
电脑浏览器下载 `https://github.com/bin336/coupon-stock/archive/refs/heads/main.zip`，用 `scp` 传到主机后解压：
```bash
# 在电脑上执行
scp ~/Downloads/main.zip root@公网IP:~/
# 回到主机
cd ~ && apt-get install -y unzip && unzip main.zip && mv coupon-stock-main coupon-stock && cd coupon-stock
```

### 改两处安全默认值（必做，和 NAS 指南一致）
打开 `docker-compose.yml`，把这两行换成你自己的：
```yaml
      - ADMIN_PASS=admin123            # ← 改成强密码
      - JWT_SECRET=change-me-...       # ← 改成一段乱码长字符串
```

### 启动
```bash
docker compose up -d --build
```
首次会联网拉 Node 镜像 + 装依赖，约 1~3 分钟。看到容器 `Up` 即成功：
```bash
docker compose ps
```

### 验证
- 主机内：`curl -s http://localhost:3000 | head` 能返回 HTML。
- 浏览器开 `http://公网IP:3000`（若开了 3000 安全组）→ 看到登录页即成功。
- 默认管理员：`admin / 你刚改的 ADMIN_PASS`。

## 六、手机在外也能访问（三种方案，二选一/三选一）

### 方案 A：直接 `公网IP:3000`（最简单，免备案）
浏览器直接开 `http://公网IP:3000`。**无需备案，当天可用。**
⚠️ 缺点：HTTP 明文传密码（文档已提醒）。若介意，用下面 B 或 C。

### 方案 B：Tailscale（加密隧道，免备案，推荐 ⭐）
和你 NAS 那套用法一致，只是把「NAS」换成「云主机」：
1. 云主机装 Tailscale：`curl -fsSL https://tailscale.com/install.sh | sh && sudo tailscale up`
2. 手机装 Tailscale App，登录**同一个账号**。
3. 云平台安全组**不用开 3000 公网**，只在 Tailscale 网络内访问：手机连上 Tailscale 后开 `http://<云主机Tailscale的100.x IP>:3000`。
   > 好处：服务不暴露公网、加密、免备案；连不上时先检查手机 Tailscale 是否连上。

### 方案 C：域名 + HTTPS（Caddy 反代，需备案，进阶）
想用 `https://你的域名` 访问时才需要，且**国内云必须 ICP 备案**（实名 + 已注册域名，审核 1~2 周）。
在 `docker-compose.yml` 里加一个 caddy 服务做反代（见下方片段），Caddy 自动申请免费证书：
```yaml
  caddy:
    image: caddy:2
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config
    depends_on:
      - coupon-stock
volumes:
  caddy_data:
  caddy_config:
```
`Caddyfile` 内容（把域名换成你的）：
```
你的域名 {
  reverse_proxy coupon-stock:3000
}
```
> 别忘了安全组放行 80 / 443，且域名已完成备案。

## 七、把 NAS 上已有的券搬过来（数据迁移）

如果你 NAS 上已经录了券，想直接带到云主机：
```bash
# 在云主机上，先把项目里的空 data 准备好，再用 scp 从 NAS 拉数据
# 假设 NAS 的 Tailscale IP 是 100.x.x.x，数据在 /share/.../coupon-stock/data
scp -r root@100.x.x.x:/share/.../coupon-stock/data ./data
docker compose restart   # 重启容器加载数据
```
> 全新部署、云主机上还没有券的话，跳过这步，直接录入即可。

## 八、备份数据回 NAS（关键，照做不丢数据）

你的全部家当在 `./data`（coupon.db + uploads/）。**云主机跑服务，NAS 当备份盘**，定时把 `data` 打包回传 NAS。

在云主机建脚本 `~/backup-to-nas.sh`：
```bash
#!/bin/bash
set -e
DATE=$(date +%Y%m%d)
SRC=~/coupon-stock/data
NAS=root@100.x.x.x:/share/.../coupon-stock-backup   # ← 换成 NAS 的 Tailscale IP 和路径
tar -czf /tmp/coupon-data-$DATE.tar.gz -C "$SRC" .
rsync -az /tmp/coupon-data-$DATE.tar.gz "$NAS"/
# 在 NAS 上只保留最近 7 份
ssh root@100.x.x.x "cd /share/.../coupon-stock-backup && ls -t | tail -n +8 | xargs -r rm -f"
echo "backup $DATE done"
```
```bash
chmod +x ~/backup-to-nas.sh
# 每天凌晨 4 点自动备份
(crontab -l 2>/dev/null; echo "0 4 * * * /root/backup-to-nas.sh >> /var/log/coupon-backup.log 2>&1") | crontab -
```
> 嫌 NAS 麻烦也可备份到阿里云 OSS / 腾讯云 COS（几毛钱/月），思路一样：`tar` 后 `ossutil cp` 上传。

## 九、日常更新代码（保数据，重点）

和 NAS 指南一致：**唯一红线是别弄丢 `data/`**。

```bash
cd ~/coupon-stock
docker compose down
cp -r data /tmp/coupon-data-backup-$(date +%Y%m%d)   # 先备份
git pull                          # 或重新传新 zip 覆盖（不要覆盖 data/）
docker compose up -d --build      # 重建，约 30 秒
cp -r /tmp/coupon-data-backup-* data   # 若 data 被动过则恢复
```
> 记住一句话：**更新动代码，不动 `data/`；动之前先备份。**

## 十、安全提醒（上线前必做）

1. `docker-compose.yml` 里改掉 `ADMIN_PASS` 和 `JWT_SECRET` 默认值，别用出厂的。
2. 首次登录立即改管理员密码（右上角头像 → 个人资料/用户管理）。
3. 若用方案 A（裸 IP:3000），尽量改用 Tailscale 或 HTTPS，避免明文传密码。
4. 主机 SSH 建议禁用密码登录、改用密钥（进阶，可选）。

## 十一、费用小结（按「续费/常规价」算，更贴近真实年支出）

| 项目 | 年费（续费/常规价） |
|------|---------------------|
| 云主机（2核2G） | ~600–1000 元 |
| 云主机（2核4G，一步到位） | ~1000–1800 元 |
| 域名（可选） | .com ~55–80 / .cn ~30–50 元 |
| 对象存储备份（可选） | 几毛~几元/月 |
| **合计（2核2G + 域名）** | **约 700–1100 元/年** |

> 首年若能抢到新人秒杀，可能只要几十~一百多元；但之后大概率回到上面的常规区间。
> 对比：一台 NAS 硬件就要 1000–5000 元（一次性）+ 电费，云主机胜在**不用买硬件、7×24 稳、随用随弃**；代价是每年有租金。按你「先 NAS 后云」的打算，NAS 先顶着、不稳再上云，正好把这笔年租金往后挪。

## 十二、常见问题

| 现象 | 可能原因 / 解决 |
|------|----------------|
| 浏览器开 `IP:3000` 连不上 | ① 云平台安全组是否放行 3000；② 系统 ufw 是否放行；③ 容器是否 `Up`（`docker compose ps`） |
| Tailscale 方案连不上 | 手机和云主机是否登录**同一** Tailscale 账号；云主机 `tailscale status` 是否正常 |
| 想用域名但打不开 443 | 国内云**必须备案**后才能用 80/443；未备案先用 IP:3000 或 Tailscale |
| OCR 识别点不动 | OCR 首次需联网下载语言包，确认云主机能上外网 |
| 数据没了 | 按第八节从 NAS 备份恢复 `data/` |
| 券的"已过期"判断不对 | 容器已固定 `TZ=Asia/Shanghai`，若仍异常检查主机时间 |
