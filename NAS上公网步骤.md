# 券库 · NAS 上公网（外网访问）详细步骤

> 目标：不在家里 WiFi（用手机流量）也能开券库，随时在微信里查券、卖券。
> **推荐方案：Tailscale**（免费、不用碰路由器、全程加密、最稳）。
> 备选方案：路由器端口映射 + DDNS（暴露到公网，新手易配错且有安全风险，不推荐）。

---

## 一、推荐：Tailscale（加密隧道，免公网暴露）⭐

### 步骤
1. **NAS 装 Tailscale 套件：**
   - 群晖：**套件中心** 搜 `Tailscale` 安装，登录一个 Tailscale 账号（用 Google / 微软邮箱注册）
   - 威联通：**App Center** 搜 `Tailscale` 安装并登录
2. **手机装 Tailscale App**（应用商店搜 Tailscale），登录**同一个账号**。
3. 手机打开 Tailscale（连上后），浏览器访问：
   ```
   http://<NAS的Tailscale IP>:3000
   ```
   Tailscale IP 在 App 里能看到（形如 `100.x.x.x`）。
4. 以后手机只要开着 Tailscale，走到哪儿都能开券库，跟你在家一样。

### 为什么推荐 Tailscale
- 不用在路由器上做端口映射、不用把 NAS 暴露到公网。
- 全程加密，比"裸 IP:3000"安全得多。
- 不用做 ICP 备案（备案是国内的云主机用 80/443 才需要，NAS + Tailscale 不涉及）。

---

## 二、备选：路由器端口映射 + DDNS（真·公网 IP 访问）

> 仅在你确实需要"任何人都能直接访问"时考虑。有安全风险，且家用宽带常被运营商封 80 端口、公网 IP 也会变动。

1. **路由器后台做端口映射（端口转发）：** 外部端口 `3000` → NAS 内网 IP `:3000`。
2. **申请动态域名（DDNS）：** 用路由器或 NAS 自带的 DDNS 功能，绑定一个域名（如 `xxx.f3322.net`、群晖的 `xxx.synology.me` 等）。
3. 浏览器开 `http://你的域名:3000` 即可。

⚠️ **注意：**
- 明文 HTTP，密码不安全；建议再加 HTTPS（NAS 反向代理 + 证书，进阶）。
- 暴露公网有被扫端口 / 爆破风险，**务必用强密码 + 改默认端口**。

---

## 三、两种方式对比

| 维度 | Tailscale | 端口映射 + DDNS |
|------|-----------|----------------|
| 难度 | 低 | 中 |
| 安全性 | 高（加密、不暴露公网） | 低（直接暴露公网） |
| 是否需要备案 | 否 | 否（但用域名 80/443 才需） |
| 稳定性 | 高 | 受公网 IP 变动影响 |

---

## 四、常见问题

| 现象 | 解决 |
|------|------|
| 手机开不了券库 | 确认手机和 NAS 都登录**同一** Tailscale 账号；手机 Tailscale 是否已连 |
| Tailscale 连上了但打不开 | 确认 NAS 上券库容器在运行、3000 端口在 NAS 内网可达 |
| 端口映射后外网连不上 | 运营商可能封 80；换高位端口；确认公网 IP 没变（DDNS 已生效） |
| 担心被外人扫到 | 用 Tailscale 方案最省心；端口映射方案务必强密码 + 改端口 |

---

> 一句话：能用 Tailscale 就用 Tailscale，安全省事；除非真要让陌生人直接访问，否则别碰端口映射。
